本包的知识点为：装饰器

1. 定义：在不改变原有函数代码的情况下，添加新的功能

2. 常用场景：判断用户是否有权限访问，判断用户是否是某个角色，测量函数运行时间，测量函数运行时间是否超过阈值（装饰器生成器：根据参数的不同，生成不同的装饰器）

   - 即常用来解决横切关注点问题，比如权限验证，日志记录和性能监控，优势是是解耦和代码复用

3. 优点：
   - 通过装饰器可以扩展别人的函数，像是调用一个库时，需要在该库基础上加一些功能，不用去该库的源代码中更改代码
   - 减少代码冗余，提高代码可复用性
   - 装饰器可以保证函数内部主逻辑清晰，这也符合软件开发中单一职责的原则

4. 注意：

   装饰器是在函数定义时就会执行，而不是在函数调用时

   ```
   @log
   def f():
       print("Function f executed")
   ```

   等价于

   ```
   def f():
       print("Function f executed")
   f = log(f)  # 装饰器在这里执行！
   ```

   所以在未写出函数调用代码   "f()"    时，装饰器就已经执行了
